UnitTestCase::UnitTestCase() - Creates an empty test case. Should be subclassed with test methods for a functional test case.

UnitTestCase::assertCopy() - Will trigger a pass if both parameters refer to different objects. Fail otherwise.

UnitTestCase::assertEqual() - Will trigger a pass if the two parameters have the same value only. Otherwise a fail.

UnitTestCase::assertError() - Confirms that an error has occoured and optionally that the error text matches exactly.

UnitTestCase::assertErrorPattern() - Confirms that an error has occoured and that the error text matches a Perl regular expression.

UnitTestCase::assertIdentical() - Will trigger a pass if the two parameters have the same value and same type. Otherwise a fail.

UnitTestCase::assertIsA() - Type and class test. Will pass if class matches the type name or is a subclass or if not an object, but the type is correct.

UnitTestCase::assertNoErrors() - Confirms that no errors have occoured so far in the test method.

UnitTestCase::assertNotA() - Type and class mismatch test. Will pass if class name or underling type does not match the one specified.

UnitTestCase::assertNotEqual() - Will trigger a pass if the two parameters have a different value. Otherwise a fail.

UnitTestCase::assertNotIdentical() - Will trigger a pass if the two parameters have the different value or different type.

UnitTestCase::assertNotNull() - Will be true if the value is set.

UnitTestCase::assertNoUnwantedPattern() - Will trigger a pass if the perl regex pattern is not present in subject. Fail if found.

UnitTestCase::assertNull() - Will be true if the value is null.

UnitTestCase::assertReference() - Will trigger a pass if both parameters refer to the same object. Fail otherwise.

UnitTestCase::assertWantedPattern() - Will trigger a pass if the Perl regex pattern is found in the subject. Fail otherwise.
